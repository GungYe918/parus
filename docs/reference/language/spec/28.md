## 16. EBNF 테이블 (v0 전체 문서 반영, 파서 제작 가능 수준)

> 표기 규칙
> `ε` = empty, `?` = optional, `*` = 0+ 반복, `+` = 1+ 반복
> `A | B` = 선택, `( ... )` = 그룹, `'tok'` = 리터럴 토큰
> **세미콜론 `;`은 “문장 종결자”**이며, `Block`은 종결자 없이 문장으로 취급된다.

---

### 16.1 Lexical (토큰 레벨)

```ebnf
Letter        := /* Unicode XID_Start (권장) 또는 구현상 IDENT_START */
LetterCont    := /* Unicode XID_Continue (권장) 또는 구현상 IDENT_CONT */

Ident         := Letter LetterCont* | BacktickIdent ;
BacktickIdent := "`" /* UTF-8 any (incl emoji) except ` and newline */ "`" ;

IntLit        := Digit (Digit | "_")* IntSuffix? ;
IntSuffix     := "i8" | "i16" | "i32" | "i64" | "u8" | "u16" | "u32" | "u64" ;

FloatLit      := Digit (Digit | "_" | ".")* FloatSuffix ;
FloatSuffix   := "f" | "lf" ;

BoolLit       := "true" | "false" ;
NullLit       := "null" ;

CharLit       := "'" CharBody "'" ;
CharBody      := /* 'a' | escape sequences | '\u{HEX+}' etc. */ ;

StringLit     := NormalString | FString | RawString | DollarFString ;
NormalString  := "\"" /* ... */ "\"" ;
FString       := "F\"\"\"" /* ... { Expr } ... */ "\"\"\"" ;
RawString     := "R\"\"\"" /* ... no interpolation ... */ "\"\"\"" ;
DollarFString := "$\"" /* ... { Expr } ... */ "\"" ;

Comment       := LineComment | BlockComment ;
LineComment   := "//" /* to end of line */ ;
BlockComment  := "/*" /* ... */ "*/" ;   /* non-nested */

WS            := /* spaces/tabs/newlines */ ;
```

### 16.2 프로그램 구조: project/bundle/module/file 단위

```ebnf
File          := (Item | EmptyItem)* EOF ;

               | ActorDecl
               | InstDecl
               | MacroDecl ;

               | GlobalVarDecl ;

DirectiveItem := ImportStmt

               | NamespaceDecl ;
```

### 16.3 경로 / 네임스페이스(nest)

```ebnf
Path          := Ident ("::" Ident)* ;

NamespaceDecl := ExportOpt "nest" Path ( ";" | BlockNamespace ) ;
BlockNamespace:= "{" NamespaceItem* "}" ;

NamespaceItem := UseStmt
               | NamespaceDecl
               | Decl
               | ";" ;
```

### 16.4 `import`/`use` 문

```ebnf
ImportStmt    := ImportModuleStmt | ImportCHeaderStmt ;
ImportModuleStmt := "import" Path ("as" Ident)? ";" ;
ImportCHeaderStmt := "import" StringLit "as" Ident ";" ;

UseStmt       := "use" UseBody ";" ;

UseBody       := UseTypeAlias
               | UsePathAlias
               | UseNestAlias
               | UseTextSubst
               | UseActsSelect
               ;

UseTypeAlias  := Ident "=" Type ;
UsePathAlias  := Path ("=" | "as") Ident ;     /* 경로 별칭 */
UseNestAlias  := "nest" Path ("as" Ident)? ;   /* namespace 별칭(= 금지) */
UseTextSubst  := Ident Expr ;                  /* IDENT 단위 치환(매크로 함수 금지) */
UseActsSelect := ActsTargetType "with" "acts" "(" ActsSetName ")" ;
ActsTargetType:= PathType ;
ActsSetName   := Ident | "default" ;
```

### 16.6 전역/정적 변수 선언

```ebnf
GlobalVarDecl    := CAbiGlobalDecl | NormalGlobalDecl ;

CAbiGlobalDecl   := LinkPrefix "static" MutOpt Ident ":" Type CAbiInitOpt ";" ;
LinkPrefix       := ("extern" | "export") "\"C\"" ;
CAbiInitOpt      := "=" Expr | ε ;              /* extern 경로에서는 ε */

MutOpt           := "mut" | ε ;
VarKw            := "let" | "set" ;

LetDecl          := Ident ":" Type "=" Expr ;
SetDecl          := Ident "=" Expr ;
```

### 16.7 함수 선언 (attribute/export/mode/qualifier/제네릭/제약/예외접미 `?`)

```ebnf
FuncDecl      := CAbiFuncDecl | CAbiFuncDef | NormalFuncDecl ;

CAbiFuncDecl  := "extern" "\"C\"" "def" FuncName FuncParams "->" Type ";" ;
CAbiFuncDef   := "export" "\"C\"" "def" FuncName FuncParams "->" Type Block ;
NormalFuncDecl:= Attribute* ExportOpt "def" ModeOpt QualifierOpt FuncName GenericParamClauseOpt FuncParams ConstraintClauseOpt "->" Type Block ;

FuncName      := Ident QMarkOpt ;
QMarkOpt      := "?" | ε ;                     /* 예외 허용 함수 표기 */

ModeOpt       := "sub" | "pub" | ε ;           /* actor 문맥에서 의미 있음 */
QualifierOpt  := ε ;                             /* v0: qualifier 키워드 문법은 보류, @attribute 경로만 문서화 */

GenericParamClauseOpt := "<" TypeParam ("," TypeParam)* ">" | ε ;
TypeParam     := Ident ;

FuncParams    := "(" ParamSectionOpt ")" ;

ParamSectionOpt :=
                 ε
               | PositionalParamsOpt NamedGroupOpt
               ;

PositionalParamsOpt :=
                 PosParam ("," PosParam)* ("," )?
               | ε ;

PosParam      := Ident ":" Type DefaultOpt ;

NamedGroupOpt := "{" NamedParamListOpt "}" | ε ;
NamedParamListOpt :=
                 NamedParam ("," NamedParam)* ("," )?
               | ε ;

NamedParam    := Ident ":" Type DefaultOpt ;

DefaultOpt    := "=" Expr | ε ;

ConstraintClauseOpt := WithBracketConstraintClause
                     | ε ;

WithBracketConstraintClause := "with" "[" ConstraintList "]" ;
ConstraintList := Constraint ("," Constraint)* ;
Constraint    := Ident ":" PathType ;
```

### 16.8 타입(Type) 문법 (nullable, 배열/리스트, borrow/slice, handle, 제네릭(파싱 선행))

```ebnf
Type          := TypePrimary TypeSuffix* ;

TypePrimary   := "(" Type ")" 
               | "handle" "<" Type ">"
               | SliceType
               | PathType
               | ArrayOrListType
               ;

PathType      := Path GenericArgsOpt ;
GenericArgsOpt:= "<" Type ("," Type)* ">" | ε ;

ArrayOrListType :=
                 PathType "[" ArrayLenOpt "]"
               | "(" Type ")" "[" ArrayLenOpt "]" ;      /* 괄호 타입도 지원 */

ArrayLenOpt   := Expr | ε ;                 /* empty => list T[] */
               /* T[N] : ArrayLenOpt=Expr, T[] : ArrayLenOpt=ε */

SliceType     := RefPrefix "[" Type "]" ;   /* &[T], &mut [T] */

RefPrefix     := "&" MutOpt ;               /* &, &mut */

TypeSuffix    := "?" ;                      /* nullable: T? (접미가 &보다 우선 결합 권장) */
```

> **주의(우선순위 정책 반영)**: `&i32?`는 문법적으로 `RefPrefix`가 `TypePrimary`의 한 형태가 아니라, `SliceType` 외에는 `&`가 “표현식 연산자”에 가까워 충돌 소지가 있으므로, 타입에서 `&T` 자체를 원한다면 아래 “RefType” 확정을 선택해도 된다. (아래 16.15 모호점 참고)

### 16.9 struct / enum / proto / class / acts / inst

```ebnf
FieldDecl     := "struct" FieldLayoutOpt FieldAlignOpt Ident GenericParamClauseOpt BlockField ";"? ;
FieldLayoutOpt:= "layout" "(" "c" ")" | ε ;
FieldAlignOpt := "align" "(" IntLit ")" | ε ;
BlockField    := "{" FieldMember* "}" ;
FieldMember   := Ident ":" Type ";" ;       /* struct 내부 함수 금지 */

EnumDecl      := "enum" FieldLayoutOpt Ident GenericParamClauseOpt ProtoInheritOpt ConstraintClauseOpt BlockEnum ";"? ;
BlockEnum     := "{" EnumCaseDeclListOpt "}" ;
EnumCaseDeclListOpt := EnumCaseDecl ("," EnumCaseDecl)* ","? | ε ;
EnumCaseDecl  := "case" Ident EnumPayloadOpt EnumDiscriminantOpt ;
EnumPayloadOpt := "(" EnumPayloadFieldListOpt ")" | ε ;
EnumPayloadFieldListOpt := EnumPayloadField ("," EnumPayloadField)* ","? | ε ;
EnumPayloadField := Ident ":" Type ;
EnumDiscriminantOpt := "=" IntLit | ε ;

ProtoDecl     := ExportOpt "proto" Ident GenericParamClauseOpt ProtoInheritOpt BlockProto ";"? ;
ProtoInheritOpt := ":" PathType ("," PathType)* | ε ;
BlockProto    := "{" ProtoItem* "}" ;
ProtoItem     := ProtoRequireKindItem
               | ProtoRequireFnItem
               | ProtoProvideFnItem
               | ProtoProvideConstItem
               | ";" ;
ProtoRequireKindItem := "require" ProtoRequireKind "(" PathType ")" ";" ;
ProtoRequireKind := "struct" | "enum" | "class" | "actor" | "acts" ;
ProtoRequireFnItem := "require" Ident QMarkOpt GenericParamClauseOpt FuncParams ConstraintClauseOpt "->" Type ";" ;
ProtoProvideFnItem := "provide" "def" Ident QMarkOpt GenericParamClauseOpt FuncParams ConstraintClauseOpt "->" Type Block ";"? ;
ProtoProvideConstItem := "provide" "const" Ident ":" Type "=" Expr ";" ;

ClassValueDecl:= ExportOpt "class" Ident GenericParamClauseOpt InheritOpt ConstraintClauseOpt BlockClassValue ";"? ;
InheritOpt    := ":" PathType ("," PathType)* | ε ;

BlockClassValue := "{" ClassValueMember* "}" ;
ClassValueMember := AccessMod
               | ClassLifecycleMember
               | ClassMethodDecl
               | ClassStaticMemberDecl
               | ClassInstanceFieldDecl
               | ";" ;

ClassMethodDecl := FuncDecl ;
ClassStaticMemberDecl := "static" ( FuncDecl | ClassStaticVarDecl ) ;
ClassStaticVarDecl := Ident ":" Type "=" Expr ";" ;
ClassInstanceFieldDecl := Ident ":" Type ";" ;

ClassLifecycleMember := InitDecl | DeinitDecl ;
InitDecl        := "init" FuncParams ClassLifecycleBody ;
DeinitDecl      := "deinit" "(" ")" ClassLifecycleBody ;
ClassLifecycleBody := Block ";"?
                    | "=" "default" ";" ;

ActsDecl      := ExportOpt "acts" ActsHead ConstraintClauseOpt BlockActs ";"? ;
ActsHead      := "for" PathType
               | Ident
               | Ident "for" PathType ;
BlockActs     := "{" ActsMember* "}" ;
ActsMember    := FuncDecl
               | OperatorDecl
               | ";" ;

OperatorDecl  := "operator" "(" OperatorName ")" FuncParams "->" Type Block ;
OperatorName  := PunctOp | "++pre" | "++post" | "copy" | "clone" ;

PunctOp       := "+" | "-" | "*" | "/" | "%" | "==" | "!=" | "<" | "<=" | ">" | ">="
               | "[]" | "()" | "++" | "=" | "+=" | "-=" | "*=" | "/=" | "%="
               | "and" | "or" | "xor" | "not" ;

InstDecl      := ExportOpt "inst" Ident InstParams Block ";"? ;
InstParams    := "(" InstParamListOpt ")" ;
InstParamListOpt := InstParam ("," InstParam)* ","? | ε ;
InstParam     := Ident ":" Type ;
```

추가 고정 규칙:

1. `acts` 제네릭은 owner 타입 표기만 허용한다.
2. 금지: `acts for Vec<T> <T> { ... }`
3. 허용: `acts for Vec<T> with [T: Proto] { ... }`
4. `struct Name<T> ...` 구문을 지원한다.
5. generic struct 리터럴은 `Name<T>{...}` 명시형만 허용한다(`Name{...}` 추론형은 v1 비지원).
6. `actor Name<T> ...` 구문은 v1에서 `GenericActorDeclNotSupportedV1`로 거부한다.
7. `proto ... with require(...)` 레거시 꼬리 문법은 제거한다.
8. `proto A: B, C`에서 `:` 뒤 대상은 `proto`로 해석되어야 한다.
9. `require` kind 항목은 `struct/enum/class/actor/acts`만 허용하며 `proto`는 허용하지 않는다.
10. proto `require/provide` 함수 시그니처는 `self` 리시버 파라미터를 금지한다.

### 16.9-A macro 선언/호출 (`with token` 기본 활성)

```ebnf
MacroDecl      := "macro" Ident "->" "{" MacroGroup+ "}" ;
MacroGroup     := "with" MatchKind "{" MacroArm+ "}" ;
MatchKind      := "expr" | "stmt" | "item" | "type" | "token" ;
MacroArm       := "(" MacroPattern ")" "=>" OutKind Block ";" ;
OutKind        := "expr" | "stmt" | "item" | "type" ;

MacroPattern   := TypedPattern | TokenPattern ;
TypedPattern   := TypedElem ("," TypedElem)* ;
TypedElem      := Ident ":" FragKind
               | Ident ":" FragKind "..." ;
FragKind       := "tt" | "expr" | "stmt" | "item" | "type" | "path" | "ident" | "block" ;

TokenPattern   := TokenElem* ;
TokenElem      := TokenLit
               | Capture
               | TokenGroup
               | Repeat ;
Capture        := Ident ":" FragKind
               | Ident ":" FragKind "..." ;
TokenGroup     := "(" TokenElem* ")" | "{" TokenElem* "}" | "[" TokenElem* "]" ;
Repeat         := "$(" TokenElem* ")" RepOp
               | "$(" TokenElem* ")" Sep RepOp ;
RepOp          := "*" | "+" | "?" ;
Sep            := "," | ";" | "|" ;

MacroCall      := "$" Path "(" ArgTokenStreamOpt ")" ;
ArgTokenStreamOpt := /* raw token stream, parser가 delimiter 균형만 확인 */ | ε ;
```

고정 규칙(v0):

1. `with token`은 기본 활성화이며 실험 플래그가 없다.
2. arm 선택은 선언 순서 first-match다.
3. 반복 본문이 no-progress면 `MacroRepeatEmpty`로 거부한다.
4. variadic 길이 불일치, 반복 문맥 외 variadic 사용은 컴파일 오류다.

### 16.9-B compiler 지시문 (`$[]` / `$![]`)

```ebnf
CompilerDirectiveItem := "$" "[" DirectiveCall "]" ItemDecl ;
CompilerIntrinsicDecl := "$" "!" "[" CompilerIntrinsicBody "]" ";" ;
CompilerIntrinsicBody := DirectivePath "=" DirectivePath
                       | DirectivePath ;
DirectiveCall := Path "(" DirectiveArgListOpt ")" ;
DirectiveArgListOpt := Expr ("," Expr)* ","? | ε ;
DirectivePath := Ident ( ("." | "::") Ident )* ;
```

고정 규칙(v1):

1. `$[]`는 **단일 item declaration**만 제어한다(블록 if/else 체인 비지원).
2. `$[If(...)]`의 `If`는 compiler builtin이며 라이브러리 제공으로 대체할 수 없다.
3. `$[if(...)]` 소문자 레거시 문법은 제거되었다.
4. 사용자 정의 지시문은 `inst Name(params) { ... }`를 선언하고 `$[Name(...)]`로 호출한다.
5. `inst`는 암묵 bool 반환이다(`-> bool` 표기 금지).
6. `inst` v1 본문은 `let`, `if/else`, `return`만 허용한다.
7. `inst` v1 식은 bool/int/string literal, ident, `not/and/or`, `==/!=`, inst 호출만 허용한다.
8. `inst` 본문/식에서 `&`, `~`, `~&`는 금지된다.
9. `$![]`는 두 형태를 지원한다.
   1. 대입형: `$![key.path = target::path];`
   2. 태그형: `$![tag.path];`
10. `$![Impl::Core];`는 파일 헤더 표식이다.
    1. top-level 첫 item 위치에서만 허용한다.
    2. 파일당 1회만 허용한다.
    3. 유효할 때 parser의 `core-file mode`를 활성화한다.
11. v1에서 `$![]`는 파싱/AST 보존 중심이며, 정책 강제(예: core 전용 규칙 완화)는 후속 tyck 단계에서 확정한다.
12. 매크로 호출 `$path(...)`와 compiler 지시문 `$[...]`, `$![...]`는 구문 계층이 다르며 상호 대체되지 않는다.

### 16.10 actor + draft/pub/sub + commit/recast (구문 레벨)

보류 참조: 아래 EBNF는 현재 구현/정본 스냅샷이다. actor 생성식 `A(...)`는 반영되었고, `async/await`는 `docs/todo/CONCURRENCY_MODEL.md`의 후속 범위를 따른다.

```ebnf
ActorDecl     := ExportOpt "actor" Ident ActorBody ";"? ;

ActorBody     := "{" ActorItem* "}" ;

ActorItem     := DraftDecl
               | ActorInitDecl
               | ActorMethodDecl
               | ";" ;

DraftDecl     := "draft" "{" DraftField* "}" ;
DraftField    := Ident ":" Type ";" ;

ActorInitDecl := "init" FuncParams ActorInitBody ;
ActorInitBody := Block ";"?
               | "=" "default" ";" ;

ActorMethodDecl := "def" ActorMode Ident FuncParams ConstraintClauseOpt "->" Type Block ;
ActorMode     := "sub" | "pub" ;

```

고정 규칙(v1):

1. actor 선언 레벨 제네릭(`actor A<T>`)은 지원하지 않는다.

### 16.12 if / while / loop / switch

```ebnf
IfExprStmt    := IfExpr ";"? ;          /* if는 표현식이지만, stmt로도 사용 가능 */

IfExpr        := "if" "(" Expr ")" Block ElifPart* ElsePartOpt ;
ElifPart      := "elif" "(" Expr ")" Block ;
ElsePartOpt   := "else" Block | ε ;

WhileStmt     := "while" "(" Expr ")" Block ;

LoopStmt      := "loop" LoopBody ;
LoopBody      := Block
               | "(" Ident "in" Expr ")" Block ;

SwitchStmt    := "switch" "(" Expr ")" "{" SwitchCase* DefaultCaseOpt "}" ;
SwitchCase    := "case" CaseLabel ":" Block ;
DefaultCaseOpt:= "default" ":" Block | ε ;

CaseLabel     := Literal
               | "null"
               | EnumCasePattern
               | Pattern ;

EnumCasePattern := PathType "::" Ident EnumCaseBindOpt ;
EnumCaseBindOpt := "(" EnumCaseBindListOpt ")" | ε ;
EnumCaseBindListOpt := EnumCaseBind ("," EnumCaseBind)* ","? | ε ;
EnumCaseBind := Ident ":" Ident ;

Pattern       := Ident "(" PatternArgsOpt ")" ;
PatternArgsOpt:= Ident
               | Ident ":" Ident
               | ε ;
```

### 16.14 표현식(Expression) (우선순위/결합 규칙 포함)

> 아래는 **프랫/우선순위 파서 제작 가능한 형태**로 계층을 분리했다.
> (v1 규칙: 삼항 `?:` 중첩 금지, `|>` 파이프는 RHS가 CallExpr이며 라벨 hole `_` 1개 강제, `<|`는 미지원)

```ebnf
Expr          := AssignExpr ;

AssignExpr    := TernaryExpr (AssignOp AssignExpr)? ;
AssignOp      := "=" | "+=" | "-=" | "*=" | "/=" | "%=" ;

TernaryExpr   := PipeExpr ("?" PipeExpr ":" PipeExpr)? ; /* v0: nested ternary 금지(검사로 제약) */

PipeExpr      := OrExpr ( "|>" PipeRhs )* ;
PipeRhs       := CallExpr ;  /* v0 강제: RHS는 CallExpr */

OrExpr        := XorExpr ( "or" XorExpr )* ;
XorExpr       := AndExpr ( "xor" AndExpr )* ;
AndExpr       := EqExpr  ( "and" EqExpr )* ;

EqExpr        := RelExpr ( ("==" | "!=") RelExpr )* ;
RelExpr       := ShiftExpr ( ("<" | "<=" | ">" | ">=") ShiftExpr )* ;
ShiftExpr     := AddExpr ( ("<<" | ">>") AddExpr )* ;

AddExpr       := MulExpr ( ("+" | "-") MulExpr )* ;
MulExpr       := PrefixExpr ( ("*" | "/" | "%") PrefixExpr )* ;

PrefixExpr    := PrefixOp PrefixExpr
               | TryExpr
               | PostfixExpr ;

PrefixOp      := "not" | "!" | "+" | "-" | "++"
               | BorrowOp
               | EscapeOp
               | CopyCloneOp
               ;

BorrowOp      := "&" MutOpt ;            /* &x, &mut x */
EscapeOp      := "~" ;                  /* ~x */
CopyCloneOp   := "copy" | "clone" ;      /* copy x, clone y */

TryExpr       := "try" PrefixExpr ;      /* Recoverable 채널을 non-? 경계에서 nullable로 포획 */

PostfixExpr   := PrimaryExpr PostfixOp* ;
PostfixOp     := CallSuffix
               | IndexSuffix
               | MemberSuffix
               | PostIncSuffix ;

CallSuffix    := CallTypeArgClauseOpt "(" ArgListOpt ")" ;
CallTypeArgClauseOpt := "<" Type ("," Type)* ">" | ε ;
IndexSuffix   := "[" Expr "]" ;
MemberSuffix  := ("." | "->") Ident ;
PostIncSuffix := "++" ;

PrimaryExpr   := Literal
               | IdentRef
               | Hole
               | ParenExpr
               | ArrayLiteral
               | LambdaExpr
               ;

ParenExpr     := "(" Expr ")" ;

IdentRef      := Path ;                  /* 변수/함수/네임스페이스/타입 경로 */
Hole          := "_" ;                    /* 파이프/호출 규칙에서 “라벨 인자 자리”로만 허용(검사 패스) */

Literal       := IntLit | FloatLit | BoolLit | NullLit | CharLit | StringLit ;

ArrayLiteral  := "[" ExprListOpt "]" ;
ExprListOpt   := (Expr ("," Expr)*)? ("," )? ;

ArgListOpt    := ArgList | ε ;
ArgList       := PosArgsOpt NamedArgsOpt ;    /* 혼합 규칙은 아래에서 제약 */

PosArgsOpt    := (Expr ("," Expr)*)? ("," )? | ε ;

NamedArgsOpt  := NamedGroupArgs | LabeledArgs | ε ;

LabeledArgs   := LabelArg ("," LabelArg)* ("," )? ;
LabelArg      := Ident ":" ExprOrHole ;
ExprOrHole    := Expr | "_" ;

NamedGroupArgs:= ("," )? "{" NamedGroupArgListOpt "}" ;
NamedGroupArgListOpt := (NamedGroupArg ("," NamedGroupArg)*)? ("," )? ;
NamedGroupArg := Ident ":" Expr ;

LambdaExpr    := CaptureList LambdaParamsOpt LambdaRetOpt Block ;
CaptureList   := "[" CaptureItemsOpt "]" ;
CaptureItemsOpt := (CaptureItem ("," CaptureItem)*)? ("," )? ;
CaptureItem   := MutOpt Ident ;

LambdaParamsOpt := "(" ParamSectionOpt ")" ;     /* v0: 예시가 [](x:int){...} 형태 */
LambdaRetOpt  := ("->" Type)? ;
```

단항 부정 의미 규칙(v1):

1. `not expr`는 bool 부정 전용이다.
2. prefix `!expr`는 builtin 정수 타입에 대한 bitwise not 전용이다.
3. `!bool`는 허용하지 않으며 `not bool_expr`를 사용해야 한다.
4. postfix `expr!`와 `expr as! T`는 위 규칙과 별개로 기존 force/unwrap 의미를 유지한다.
5. `.`는 concrete 멤버(field/class/actor/acts method) 전용이다.
6. `->`는 proto `provide` 멤버 전용이다.
7. `x->name` 후보가 여러 proto에서 충돌하면 `x->Proto.name`을 강제한다.

제네릭 호출 파싱/오류 정책(v1.1):

1. `foo<T>(...)`, `obj.m<T>(...)`, `ns::f<T>(...)`를 지원한다.
2. `<...>`가 호출 타입 인자 문맥으로 확정되지 않으면 비교식 파싱으로 안전 롤백한다.
3. 제네릭 경로 오류는 단일 핵심진단 우선으로 보고한다.

호출 해석 규칙(v0):

1. `A(...)`에서 `A`가 class 타입 경로면 생성식으로 해석하며 `A::init(...)` 오버로드를 대상으로 매칭한다.
2. `a.m(...)` dot 호출은 첫 파라미터가 `self`인 메서드 후보만 허용한다.
3. class/proto 멤버 경로 호출은 기본적으로 제거한다. 단, `static def`로 선언된 class 정적 멤버는 `A::m(...)` 경로 호출을 허용한다.
4. `T::acts(Name)::member(...)` 및 일반 namespace 경로 호출은 유지한다.

### 16.15 Place Expression (place 제약: `&`, `~`, delete 대상)

```ebnf
PlaceExpr     := PlaceAtom PlaceSuffix* ;

PlaceAtom     := IdentRef | "(" PlaceExpr ")" ;

PlaceSuffix   := "." Ident
               | "[" Expr "]" ;
```

의미 제약(v0):

1. `&` / `&mut`는 place expression에만 적용한다.
2. `&` / `&mut` 피연산자는 owned place여야 하며, `&T`/`&mut T`/`~T` 값에는 재적용할 수 없다.
3. `~`는 place expression에만 적용하며, v0에서 루트 식별자 move(`~x`)만 허용한다.
