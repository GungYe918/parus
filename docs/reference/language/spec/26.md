## 14. 구현 체크리스트 (v0)

### 14.1 프리패스

1. 매크로 확장
2. import/use 해석
3. bundle 단위 선언 수집 및 의존 그래프 구성
4. 타입 이름 해석(type resolve)

### 14.2 파서

1. `field/proto/class/acts/actor` 선언 파싱
2. actor 전용 구문 파싱
   1. `draft { ... }`
   2. `init(...)` / `init() = default;`
   3. `def sub ...`, `def pub ...`
   4. `spawn A(...)`
   5. `commit;`, `recast;`
3. actor 본문 금지 규칙 파싱 진단
   1. `let/set/mut/static/deinit` 금지
   2. `def foo`(mode 없음) 금지

### 14.3 타입체커

1. actor 선언 검증
   1. draft 블록 정확히 1개
   2. pub top-level commit 최소 1개
2. 컨텍스트 제약
   1. `commit`은 pub 내부 전용
   2. `recast`는 sub 내부 전용
3. 호출 규칙
   1. actor 생성은 `spawn A(...)`만 허용
   2. actor 메서드 호출은 dot 호출만 허용 (`a.m(...)`)
   3. `Actor::m(...)` path call 금지
4. lifecycle 직접 호출 금지
5. `&&draft` move 금지

### 14.4 IR 하향(SIR/OIR/LLVM)

1. actor draft 레이아웃을 field 메타로 하향
2. actor 메서드의 sub/pub mode 정보를 심볼 맹글링에 반영
3. `spawn`을 init 호출 경로로 하향
4. `commit/recast`를 OIR marker inst로 하향
5. LLVM에서 actor marker helper call로 방출
