## 7.7 loop 형태별 상세 규칙

### 7.7.1 무한 루프: `loop { ... }`

* 자연 종료 경로 없음
* `break expr;` 또는 `break;`로만 종료 가능
* `break;` 사용 시 결과는 `null`

예시:

```parus
def retry() -> int? {
  set attempts = 0;
  loop {
    attempts = attempts + 1;
    if (attempts >= 3) { break 42; }
  }
}
```

---

### 7.7.2 순회 루프: `loop (name in iterable) { ... }`

#### 허용 source (current model)

* semantic entry는 `iter::Iterator` / `iter::IterSource*` 프로토콜이다.
* `loop (name in source)`는 source expression을 직접 순회하지 않고,
  source acquisition mode에 따라 iterator를 얻어 반복한다.
* 현재 source acquisition은 다음 세 갈래다.
  * bare `expr`      -> by-value (`IterSourceMove`)
  * `&expr`          -> shared (`IterSource`)
  * `&mut expr`      -> mutable (`IterSourceMut`)
* 배열 `T[N]`와 slice view `T[]`는 현재 단계에서만 compiler-owned adapter로 유지한다.
* `a..b`, `a..:b`는 loop 전용 특수문이 아니라 ordinary `Range` 값 표현으로 취급한다.
* `text` direct iteration은 아직 열지 않는다.

#### 범위 규칙

* `a..b`, `a..:b`는 각각 `Range<T>`, `RangeInclusive<T>` 값으로 타이핑된다.
* 값 계층 helper는 `constraints::Comparable`을 기준으로 정의한다.
* 실제 순회 bridge는 `constraints::Step`을 요구한다.
* 현재 `Step`은 discrete successor를 가지는 primitive family(`i*`, `u*`, `isize`, `usize`, `char`)에만 열린다.
* 한쪽 bound가 infer-int이고 다른 쪽이 concrete integer이면 concrete 쪽으로 해석한다.
* 양쪽 bound가 모두 infer-int이면 loop header에서만 `i32`로 해석한다.

#### 반복 변수

* 타입은 iterable의 요소 타입
* 요소 타입은 source가 구현한 iterator의 `Item` associated type이다.
* `loop` 자체는 값/공유 borrow/가변 borrow를 추론하지 않는다.
  어떤 item이 나오는지는 source protocol 구현이 결정한다.
* 반복 변수는 현재 ordinary binder 하나만 허용한다.

예시:

```parus
def sum_arr(xs: int[4]) -> int {
  set mut s = 0;
  loop (v in xs) {
    s = s + v;
  }
  return s;
}
```

```parus
def find_positive(xs: int[]) -> int? {
  loop (v in xs) {
    if (v > 0) { break v; }
  }
}
```

```parus
def find_char() -> char? {
  loop (ch in 'a'..:'z') {
    if (ch == 'k') { break ch; }
  }
}
```
