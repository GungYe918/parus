# 01. Goals And Non-Goals

## 목표

1. Parus 빌드 모델(`file -> module -> bundle -> project`)을 선언형으로 기술한다.
2. TOML/YAML보다 높은 표현력(조건식, 병합, 오버레이)을 제공한다.
3. 빌드 입력이 결정적이며 재현 가능하도록 평가 규칙을 강제한다.
4. 빌드 그래프 생성(Ninja lowering)에 필요한 정보를 구조적으로 제공한다.

## 비목표

1. 일반 목적 스크립트 언어가 되는 것
2. IO 중심 자동화 언어가 되는 것
3. Turing-complete 계산 모델 제공
4. 초기 버전에서 표준 라이브러리 생태계를 여는 것

## 비튜링 정책 (v0 고정)

1. `for`, `while` 금지
2. 재귀 호출 금지
3. effect/IO 금지
4. 동적 코드 실행 금지
5. 평가 예산 초과 시 즉시 실패

## 문법 방향

1. LEI 기존 감성 유지
2. 들여쓰기 블록 폐지
3. 블록은 `{}`
4. 배열은 `[]`
